## 1. 目的

这份文档用于固定降重复率算法包训练的长期共识，避免后续版本迭代时再次把目标理解偏。

## 2. 当前核心目标

降重复率算法包的核心目标不是“把原文改得面目全非”，而是：

1. 改写后提交到目标平台复检，重复率尽量压到 `5%` 以内
2. 不能改变原文核心意思
3. 不能改坏事实、数据、术语、引文
4. 不能改出病句、错别字、断裂逻辑
5. 不能把文章改得像机器洗稿

## 3. 当前固定原则

后续任何平台、任何版本的降重复率训练，都必须坚持：

1. 真实平台复检结果优先于离线估计
2. 原意保持优先于表面降重幅度
3. 结构重写优先于机械换词
4. 保护项冻结优先于激进改写
5. 单篇偶然成功不能代替整套 benchmark 通过

## 5. 资料优先级

对降重复率最有价值的样本，不是孤立原文，而是成套对照样本。

优先级从高到低：

1. 原文 + 改写稿 + 改写前报告 + 改写后报告
2. 同一篇文章多个改写版本，对应多个真实平台结果
3. 带全文对照 / 标明引文 / 简洁报告的完整查重资料
4. 只有原文和部分报告的资料
5. 只有原文，没有真实报告的资料

规则：

1. 能直接进入评分 benchmark 的，必须优先是成套对照样本
2. 不成套的资料只能先进入规则抽取池

## 6. 当前评价结论必须围绕 4 个问题

以后评估任何降重复率候选包，都先回答下面 4 个问题：

1. 真实平台复检率有没有进 `<= 5%`
2. 原文高风险片段有没有被真正打散
3. 原意和事实有没有保持住
4. 文章语言质量有没有守住

这 4 个问题里，任何一个答不上来，都不能推进主线升级。

## 7. 当前已固化的红线

后续训练不要再重复踩这些坑：

1. 机械同义词替换
2. 为了降重删掉关键论证
3. 把专有名词、数字、政策名改错
4. 为了表面变化强行口语化
5. 句子被撑长，但逻辑更差
6. 摘要、正文、结语前后不一致
7. 整体读感像机器洗稿

## 8. 当前训练方向

降重复率训练应重点推进以下方向：

1. 平台特有高风险句法骨架提取
2. 长重合片段拆解与重组
3. 保护项识别与冻结
4. 改写后语言质量清洗
5. 改写后结构完整性校验

## 9. 平台化训练原则

后续不要用“一套 dedup 规则通吃三平台”的思路直接替代平台训练。

原则：

1. 知网单独评分、单独晋级
2. 维普单独评分、单独晋级
3. PaperPass 单独评分、单独晋级

可以共享基础改写能力，但不能共享最终评分结论。

## 10. 当前结论

以后只要说“降重复率算法包升级”，默认指的是：

1. 有真实平台结果支撑
2. 有原意保持约束
3. 有质量红线约束
4. 有独立评分门槛

如果没有这 4 条，就不算真正的升级，只能算局部试验。
