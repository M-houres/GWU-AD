# 降重复率算法包评价框架

参见：

- `docs/ALGO_PACKAGE_TRAINING_WORKSPACE.md`
- `docs/DEDUP_ALGO_TRAINING_MEMORY.md`
- `docs/DEDUP_REWRITE_REQUIREMENTS.md`
- `docs/VIP_DEDUP_SAMPLE_RULESET.md`

## 1. 目标

这份框架用于评价 `降重复率` 算法包，核心目标只有一个：

对同一篇文章完成改写后，再提交到目标平台复检，重复率结果要尽量压到 `5%` 以内，同时不能破坏原文的原意、逻辑连贯性、语言质量和学术可读性。

这里的评价重点不是：

1. 我们本地估算的重复风险看起来低不低
2. 改写后表面上换了多少词
3. 报告样式像不像目标平台

这里真正要看的，是：

1. 目标平台真实复检结果是否达标
2. 高风险重合片段是否真的被打散
3. 文章的意思、事实、数据、引用、术语有没有被破坏
4. 改写后的成文质量是否仍然合格

## 2. 适用范围

本框架适用于：

1. `知网降重复率`
2. `维普降重复率`

对应内部算法包槽位：

1. `cnki_dedup_v*.zip`
2. `vip_dedup_v*.zip`

## 3. 北极星定义

一个真正合格的降重复率算法包，最终状态应该同时满足下面 6 点：

1. 改写后提交到目标平台，重复率 `<= 5%`
2. 原始高风险重合片段被大幅打散，而不是只做表层换词
3. 论文核心观点、论证方向、结论、数据口径不变
4. 关键实体、数字、政策名、术语、引文不被改坏
5. 文章语言自然、连贯、无明显语病、无错别字
6. 输出稳定，不出现漏段、乱序、占位符、半成品

## 4. 硬门槛

任何候选版本，只要触发下面任一条，就不能升级为主线版本：

1. 目标平台真实复检重复率 `> 5%`
2. 存在明显原意偏移、论点反转、结论走样
3. 存在关键事实错误：
   - 数字错
   - 时间错
   - 实体名错
   - 政策法规名错
   - 引文标题错
4. 存在明显语言质量问题：
   - 严重病句
   - 明显错别字
   - 大段不通顺
   - 上下文跳脱
5. 存在结构性缺陷：
   - 漏段
   - 标题层级错乱
   - 段落顺序异常
   - 参考文献、数据表述被误改

补充说明：

`<= 5%` 是结果门槛，不是建议值。离线预测分再漂亮，只要真实平台复检没有压到目标区间，就不能算升级成功。

## 5. 参考样本要求

没有成套样本，就无法做可靠评价。

每个降重复率 benchmark 样本，最好至少包含：

1. 原文 `docx`
2. 改写前目标平台查重报告
3. 改写后版本
4. 改写后目标平台查重报告
5. 原文与改写稿的一一对应关系
6. 必要时补一份人工核查说明

推荐标准化字段：

```json
{
  "sample_id": "vip_dedup_doc_001",
  "platform": "vip",
  "function_type": "dedup",
  "source_text": "...",
  "rewrite_text": "...",
  "reference": {
    "before_report": {
      "overall_rate_pct": 27.33,
      "high_risk_spans": [],
```

## 6. 评分维度

总分：`100`

### 6.1 目标平台复检结果: 35 分

这是最高权重。

核心看：

1. 改写后真实重复率是否进入 `<= 5%`
2. 如果没进 `5%`，距离目标还有多远

推荐评分：

```text
if after_rate <= 5:
    score_rate = 100
elif after_rate >= 15:
    score_rate = 0
else:
    score_rate = 100 * (1 - (after_rate - 5) / 10)
weighted_rate = score_rate * 0.35
```

解释：

1. `5%` 及以下拿满这一项
2. `15%` 及以上这一项记 `0`
3. `5%-15%` 之间按线性衰减

### 6.2 高风险片段消减效果: 20 分

不是只看最终总重复率，还要看原来被标红的片段有没有真的被打散。

建议看 3 个子指标：

1. 高风险字符消减率
2. 长重合片段保留惩罚
3. 旧高风险段落残留率

推荐公式：

```text
fragment_reduction = 1 - (after_high_risk_chars / max(before_high_risk_chars, 1))
retained_block_penalty = retained_long_overlap_chars / max(before_high_risk_chars, 1)
paragraph_cleanup = cleaned_risk_paragraphs / max(before_risk_paragraphs, 1)

score_fragment =
    0.5 * fragment_reduction +
    0.3 * paragraph_cleanup +
    0.2 * (1 - retained_block_penalty)

weighted_fragment = score_fragment * 100 * 0.20
```

### 6.3 原意保持与论证一致性: 20 分

降重复率不是重写一篇新论文，而是在不改变原意的前提下降低重合。

这部分重点核查：

1. 研究对象是否没变
2. 论点是否没变
3. 因果关系是否没变
4. 结论是否没变
5. 正负态度、比较关系、范围边界是否没变

建议分为 4 个小项：

1. 主题与研究对象一致
2. 核心论点一致
3. 论证链条一致
4. 结论一致

任一项出现重大偏差，直接判为失败样本。

### 6.4 关键事实与术语保护: 10 分

这一项单独拉出来，是因为很多“降重成功”的低质改写，实际上是把事实改坏了。

重点保护对象：

1. 人名、地名、机构名、公司名
2. 数字、比例、时间、面积、金额
3. 政策名、法规名、专有术语
4. 引文书名、论文名、表格名
5. 数据口径、变量口径、样本范围

建议做成 `protected_items` 白名单。

命中规则：

1. 严重错误 `1` 个就判失败
2. 轻微格式改动但不影响事实，可酌情扣分

### 6.5 语言质量: 10 分

即使复检率达标，如果改写稿读起来像机器残稿，也不能升级。

重点检查：

1. 语法是否通顺
2. 是否存在重复赘述
3. 是否有错别字
4. 是否有断句异常
5. 是否出现明显口语化、拼接感、机械改写感

推荐规则：

1. 严重病句 `0` 容忍
2. 明显错别字 `0` 容忍
3. 轻微不顺句允许少量存在，但不能集中爆发

### 6.6 结构完整性与学术风格: 5 分

降重复率不能把论文改散。

这部分检查：

1. 标题结构是否保留
2. 段落顺序是否稳定
3. 摘要、引言、正文、结语是否完整
4. 学术表达风格是否保持

## 7. 升级评分等级

建议按下面的等级判断候选包质量：

### S 级

1. `pass@5 >= 90%`
2. 平均复检率 `<= 4.5%`
3. 原意保持均分 `>= 95`
4. 严重事实错误 `0`
5. 严重语病 / 错别字 `0`

### A 级

1. `pass@5 >= 80%`
2. 平均复检率 `<= 5.0%`
3. 原意保持均分 `>= 92`
4. 严重事实错误 `0`
5. 严重语病 / 错别字 `0`

### B 级

1. `pass@5 >= 60%`
2. 平均复检率 `<= 8.0%`
3. 原意保持均分 `>= 88`
4. 允许少量轻微表达问题，但不允许严重错误

### C / D 级

1. 无法稳定压到目标区间
2. 高风险片段残留明显
3. 原意或语言质量经常受损

主线升级要求：

1. 至少达到 `A`
2. 所有硬门槛全部通过
3. 与当前主线版本相比有明确提升，不允许核心维度退化

## 8. 训练思路

降重复率算法包训练，不应靠“同义词替换越多越好”，而应按下面的流程推进。

### 8.1 资料分层

同一平台的资料，拆成 3 层：

1. 结果评分集
   - 有原文
   - 有改写稿
   - 有改写前后真实平台报告
2. 规则抽取语料
   - 有原文和多个版本
   - 但报告不完整或配对不全
3. 保护项语料
   - 用来抽取不能误改的实体、数字、术语、引文

### 8.2 先抽“高风险骨架”，再写改写规则

重点不是找词，而是找被平台反复打红的句法骨架。

常见高风险骨架包括：

1. `对……进行研究 / 分析`
2. `从……方面提出对策`
3. `分析现状 - 发现问题 - 提出建议`
4. 长串并列定语 + 固定结论句
5. 带行业背景数据的模板开头
6. 案例描述中的整块事实链复制

训练重点：

1. 先定位这些骨架
2. 再设计结构改写
3. 最后才做少量词级替换

### 8.3 改写策略顺序

建议按以下顺序处理：

1. 保护关键实体和数字
2. 打散长重合链条
3. 重排分句顺序
4. 把模板句改成任务驱动句
5. 对过于固定的结论句做结构重写
6. 最后做语言清洗，去掉病句和机械感

### 8.4 平台差异要分开训练

不同平台的高风险判断口径不同，不能共用一个通吃规则集。

要求：

1. 知网单独训练
2. 维普单独训练

同样一句话，在不同平台的查重表现可能不同，所以：

1. 规则可以共享一部分基础能力
2. 评分、晋级、主线发布必须分平台

### 8.5 双阶段评测

为了控制时间和成本，建议采用两阶段评测：

第一阶段：离线预筛

1. 看保护项是否被破坏
2. 看句法骨架是否被打散
3. 看是否出现病句、错别字、漏段

第二阶段：真实平台复检

1. 对通过预筛的候选版本送目标平台复检
2. 以真实平台报告为最终结果

原则：

离线评分只能做预筛，不能代替真实平台复检。

### 8.6 训练闭环

每轮训练都按下面闭环执行：

1. 从平台资料包里整理新样本
2. 去重并标注样本质量
3. 生成或更新 benchmark
4. 训练候选规则包
5. 先跑离线质量门
6. 再跑目标平台复检
7. 记录每个样本的前后重复率变化
8. 记录失败原因：
   - 没压下去
   - 原意偏了
   - 事实错了
   - 语言坏了
9. 只针对主要失败原因调参
10. 回归 benchmark

## 9. 禁止做法

后续训练中，禁止以下做法：

1. 只做机械同义词替换
2. 为了降重故意改动原始事实
3. 为了过检测删除关键论证
4. 用口语化、低学术感表达替代原文
5. 只看最终重复率，不看语言质量和原意保持
6. 用单篇偶然成功样本替代整套 benchmark 评估

## 10. 当前结论

降重复率算法包的核心评价顺序，必须固定为：

1. 目标平台真实复检是否 `<= 5%`
2. 原意和事实是否保持
3. 高风险重合片段是否真的被消掉
4. 语言质量是否仍然合格

这 4 条里，任何一条不成立，都不能称为真正可升级的降重复率算法包。
