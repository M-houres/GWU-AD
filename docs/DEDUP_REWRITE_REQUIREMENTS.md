# 降重复率改写质量要求

参见：

- `docs/DEDUP_RESULT_EVAL_FRAMEWORK.md`
- `docs/DEDUP_ALGO_TRAINING_MEMORY.md`
- `docs/VIP_DEDUP_SAMPLE_RULESET.md`

## 1. 核心原则

降重复率不是“把文章改得面目全非”，而是在保持论文可用性的前提下，消解目标平台判定的高风险重合结构。

优先级顺序必须固定为：

1. 原意不变
2. 事实不坏
3. 结构不散
4. 语言不差
5. 在上述前提下，把目标平台重复率压低

结论：

如果某种改写方法虽然能降重复率，但会让文章变得乱、空、假、僵、错，那这种方法不能进入正式算法包。

## 2. 什么叫“改乱了”

后续评估中，下面这些情况，都视为“改乱了”：

1. 研究对象被改掉
2. 论点被改偏、改反、改空
3. 事实数字、时间、名称被改错
4. 原本一段完整论证，被拆得支离破碎
5. 原本学术化的句子，被改成口语化表达
6. 为了避重合，强行堆砌同义词，读起来别扭
7. 段落衔接断裂，上下文跳
8. 标题、摘要、结语之间不再一致
9. 引文、术语、专有名词被误改
10. 看起来像机器洗稿，而不是正常学术改写

## 3. 必须保留的内容

降重复率改写中，以下内容默认视为“保护项”，不能随意动。

### 3.1 研究信息

1. 题目中的研究对象
2. 核心研究主题
3. 研究范围
4. 研究结论

### 3.2 事实信息

1. 年份、时间、比例、数量、金额、面积
2. 人名、校名、公司名、机构名、地名
3. 具体政策、法规、标准、制度名称
4. 数据口径、样本口径、变量口径

### 3.3 学术信息

1. 书名号内标题
2. 引文作者和作品名
3. 专业术语
4. 表格、图表、指标名

### 3.4 结构信息

1. 摘要
2. 关键词
3. 引言
4. 正文章节层级
5. 结语 / 结论
6. 参考文献

## 4. 允许的改写方向

降重复率允许改，但要按“结构重写优先、词级替换靠后”的原则进行。

### 4.1 允许做的事

1. 调整分句顺序
2. 把长句拆成两句或三句
3. 把固定模板句改成更自然的任务驱动句
4. 把名词堆叠句改成动词驱动句
5. 把并列罗列句改成层次更清楚的表达
6. 在不改变结论的前提下，改写论证展开方式
7. 对重复性很强的套话做适度去模板化
8. 增加必要的逻辑连接，让改写后更顺

### 4.2 推荐优先做的事

1. 先打散整块高重合链条
2. 再重组句法结构
3. 然后检查保护项是否被破坏
4. 最后做语言润色

## 5. 不允许的改写方式

### 5.1 禁止型

1. 机械同义词替换
2. 为了降重删掉关键论据
3. 把结论改弱、改空、改模糊
4. 编造原文没有的新事实
5. 删除数据、案例、引文来换取低重复率
6. 误改专有名词、政策名、术语
7. 把学术表达改成明显口语
8. 用大量废话把句子“撑开”
9. 保留原句骨架，只换几个词
10. 大面积改写后前后风格不统一

### 5.2 高风险型

这些方法不是绝对禁止，但默认高风险，必须谨慎：

1. 大幅增删段落
2. 改动章节标题
3. 改动摘要与结语的中心判断
4. 把定义句、概念句改写得过于自由
5. 改动带数字和比较关系的句子

## 6. 合格改写应具备的文本特征

一个合格的降重复率改写稿，读起来应该像“同一作者重新组织表达”，而不是“另一篇乱写的新文”。

应具备以下特征：

1. 主题清楚
2. 句子顺畅
3. 逻辑自然
4. 语气稳定
5. 章节承接正常
6. 学术感仍在
7. 不出现明显机器拼接感

## 7. 改写质量检查清单

每篇改写稿至少检查下面 8 项：

1. 题目和研究对象是否没变
2. 摘要和结语是否仍然一致
3. 核心数据和专有名词是否没错
4. 章节层级和段落顺序是否完整
5. 高风险重合句是否真的被打散
6. 有无病句、错别字、重复赘述
7. 是否仍保持学术语气
8. 是否存在“为了降重而硬改”的痕迹

只要其中任一项明显失真，就不能认为这篇改写是合格结果。

## 8. 建议的改写强度控制

降重复率不是改得越狠越好。

建议控制原则：

1. 优先重写高风险句，不要无差别重写全文
2. 优先改模板化骨架，不要优先改实体词
3. 优先解决整块重合，不要只处理零碎词语
4. 能通过结构变化解决的，不靠胡乱扩写解决
5. 能保持原文学术风格的，不往口语方向偏

## 9. 训练落地要求

后续算法训练时，这份文档要直接转化为质量门：

1. 保护项识别
2. 禁止型写法拦截
3. 语言质量清洗
4. 学术风格回正
5. 结构完整性检查

也就是说，降重复率算法包不能只做“降”，还必须做“护”：

1. 护原意
2. 护事实
3. 护结构
4. 护语感

## 10. 当前要记住的结论

后续不要再把降重复率理解成“改得不像原文就行”。

正确理解应该是：

1. 目标平台复检率要低
2. 但文章质量不能掉
3. 如果只能靠乱改才能降下去，那说明算法包能力还不够，不是文章该为算法让路
