# AIGC 算法训练记忆

最后更新：2026-04-10

## 1. 目的

这份文档用于固化 AIGC 检测算法训练的长期共识，避免后续版本迭代时重复遗忘以下内容：

1. 我们评估的是“检测结果是否接近真实平台”，不是报告样式是否像真实平台。
2. 算法包升级必须接受量化评分约束，不能只凭感觉替换。
3. 原始训练资料、标准化 benchmark、离线评分结果要分层管理。
4. 当前保留的两个平台都要按同一套资料清单和评分口径推进。

## 3. 评分口径摘要

总分：100 分

评分维度：

1. 全文总分一致性：25 分
2. 风险结构一致性：20 分
3. 段落对齐：25 分
4. 高亮片段对齐：25 分
5. 稳定性：5 分

当前硬门槛：

1. 平均全文分差 `<= 10`
2. `high + medium` 风险文字占比分差 `<= 12`
3. 风险段落召回率 `>= 0.75`
4. 高亮片段重合 `F1 >= 0.65`
5. 高亮严重度匹配率 `>= 0.75`
6. 重复运行分数方差 `<= 1.0`

当前晋级规则：

1. `final_score >= 85`
2. 所有硬门槛通过
3. 与当前激活版本相比至少有明确提升，不能出现核心维度明显退化

结论解释：

- `S / A`：可以考虑主线晋级
- `B`：可用，但与真实平台还不够接近
- `C / D`：不能作为主推版本

### 4.2 当前知网第一轮对照结果

对照对象：当前 `ProcessingEngine` 的 `cnki` 检测输出

结果摘要：

1. 样本 A：真实 `18.1%`，当前 `18.77%`，差 `0.67`
2. 样本 B：真实 `27.6%`，当前 `20.77%`，差 `6.83`
3. 样本 C：真实 `0.0%`，当前 `21.05%`，差 `21.05`
4. 样本 D：真实 `32.5%`，当前 `20.51%`，差 `11.99`

当前阶段结论：

1. 全文分数不是完全失真，但区分度不够，容易压到 `20%` 左右。
2. 最大问题不是“总分差一点”，而是“风险结构不准”和“高亮命中差”。
3. 对 `0%` 文档的抑制能力明显不足，存在偏高误报。

当前评分等级：

1. 4 个样本全部为 `D`
2. 主要丢分项：
   - 风险结构一致性
   - 高亮片段对齐
3. 通过项相对较多的是：
   - 段落召回
   - 稳定性

### 4.3 当前优先优化方向

知网 AIGC 近期优先级：

1. 先解决 `0%` 样本被打到 `20%+` 的误报问题。
2. 拉开不同文档之间的总分区分度，避免总分集中在 `20%` 附近。
3. 调整 `high / medium / low / clean` 风险带比例。
4. 提高高亮命中，尤其是摘要、引言、模板化段落。
5. 长期需要补显式 `start / end` 偏移，减少近似比对误差。

## 9. 标准训练流程

每次做 AIGC 检测包升级，都按以下顺序：

1. 收集本地原始资料：
   - 原文
   - 完整真实报告
   - 版本映射
2. 先转 benchmark：
   - 生成 `reference JSON`
3. 跑当前候选算法：
   - 生成 candidate 结果
4. 跑离线评分器：
   - 看总分
   - 看 5 个维度
   - 看硬门槛
5. 只针对主要丢分项调参/调规则
6. 回归跑 benchmark
7. 达到晋级线后，才允许替换主线版本

禁止做法：

1. 没 benchmark 直接换主线包
2. 只看单篇效果，不看基准集
3. 只看总分，不看风险结构和高亮

## 10. 当前要记住的核心结论

后续不要遗忘以下事实：

1. 报告样式不是当前核心目标，结果一致性才是核心目标。
2. 当前知网 AIGC 的主要问题不是段落召回，而是：
   - 风险结构
   - 高亮片段命中
   - `0%` 文档误报
3. 未来任何算法训练都必须回到 benchmark 和评分器，不再靠主观判断。
