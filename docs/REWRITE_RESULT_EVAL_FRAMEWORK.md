# 降 AIGC 算法包评价框架

参见：

- `docs/ALGO_PACKAGE_TRAINING_WORKSPACE.md`
- `docs/REWRITE_ALGO_TRAINING_MEMORY.md`
- `docs/REWRITE_QUALITY_REQUIREMENTS.md`
- `docs/CNKI_REWRITE_SAMPLE_RULESET.md`

## 1. 目标

这份框架用于评价用户侧 `降 AIGC` 算法包。

内部函数类型仍记为 `rewrite`，但评价目标已经固定：

对同一篇文章完成改写后，再提交到对应官方平台进行 AIGC 检测，检测率要尽量压到 `5%` 以内，同时不能把文章改得乱、假、散、错。

这里真正要评价的不是：

1. 文本表面改动幅度大不大
2. 词换得多不多
3. 看起来像不像“人工写的”
4. 我们本地预估的风险值低不低

这里真正要看的，是：

1. 官方平台复检 AIGC 结果是否达标
2. 原文高风险表达是否被真正消解
3. 原意、结论、事实、论证有没有保持住
4. 改写稿是否仍然是正常、自然、可用的学术文本

## 2. 适用范围

本框架适用于：

1. `知网降 AIGC`
2. `维普降 AIGC`

对应内部算法包槽位：

1. `cnki_rewrite_v*.zip`
2. `vip_rewrite_v*.zip`

## 3. 北极星定义

一个真正合格的降 AIGC 算法包，最终状态要同时满足下面 6 点：

1. 改写后提交到官方平台复检，AIGC 检测率 `<= 5%`
2. 原本容易被判为 AIGC 的模板化表达、压缩式表达、机械逻辑链被明显消解
3. 论文核心观点、研究对象、论证方向、结论保持一致
4. 数据、术语、政策名、引用、实体名不被改坏
5. 改写后的文本仍然连贯、通顺、有学术感
6. 输出稳定，不出现漏段、断裂、口语化失控、机器拼接感

## 4. 硬门槛

候选版本只要触发下面任一条，就不能升级为主线版本：

1. 官方平台复检 AIGC 率 `> 5%`
2. 核心论点、结论、态度、研究对象明显偏移
3. 关键事实或关键术语被改错：
   - 数字错
   - 时间错
   - 实体名错
   - 政策名错
   - 引文标题错
4. 语言质量明显不合格：
   - 病句
   - 错别字
   - 上下文跳跃
   - 明显口语化
   - 机械洗稿感重
5. 结构性损坏：
   - 漏段
   - 章节顺序错乱
   - 摘要、正文、结语不一致
   - 标题层级被破坏

补充说明：

`<= 5%` 是结果门槛，不是参考建议。离线预估只能做预筛，不能代替官方平台复检。

## 5. 参考样本要求

每个降 AIGC benchmark 样本，最好至少包含：

1. 原文 `docx`
2. 改写前官方平台 AIGC 报告
3. 改写稿
4. 改写后官方平台 AIGC 报告
5. 原文与改写稿的一一对应关系
6. 必要时补一份人工核查说明

推荐标准化结构：

## 6. 评分维度

总分：`100`

### 6.1 官方平台复检结果: 35 分

这是最高优先级。

核心看：

1. 改写后 AIGC 率是否进 `<= 5%`
2. 如果没有达标，距离目标还有多远

推荐评分：

```text
if after_rate <= 5:
    score_rate = 100
elif after_rate >= 20:
    score_rate = 0
else:
    score_rate = 100 * (1 - (after_rate - 5) / 15)
weighted_rate = score_rate * 0.35
```

### 6.2 降幅效果: 15 分

不仅要看最终结果，还要看相比原文是否真的下降明显。

建议看：

1. 绝对降幅
2. 相对降幅

推荐公式：

```text
absolute_drop = max(0, before_rate - after_rate)
relative_drop = absolute_drop / max(before_rate, 1)

score_drop =
    0.6 * min(1, absolute_drop / 20) +
    0.4 * min(1, relative_drop / 0.75)

weighted_drop = score_drop * 100 * 0.15
```

说明：

如果原文本身就不高，这一项不应该单独决定晋级，但它能反映算法包是否真的在消解 AIGC 风险。

### 6.3 高风险表达消解效果: 15 分

降 AIGC 不是简单换词，而是要消解平台容易识别的表达模式。

重点检查：

1. 模板化句群是否被打散
2. 机械逻辑链是否被自然化
3. 压缩名词堆叠是否被展开
4. 高疑似段落是否仍然整段保留原骨架

这一项建议结合：

1. 官方报告高风险段落变化
2. 平台特有风险骨架命中数变化
3. 改写前后句法骨架相似度下降幅度

### 6.4 原意保持与论证一致性: 20 分

降 AIGC 不允许“为了像人写的而把论文写歪”。

重点核查：

1. 研究对象一致
2. 核心论点一致
3. 因果关系一致
4. 结论一致
5. 论证链条仍然完整

任一项出现重大偏差，直接判失败样本。

### 6.5 关键事实与保护项: 10 分

重点保护对象：

1. 人名、地名、机构名、学校名、公司名
2. 年份、时间、比例、金额、样本量、面积
3. 政策名、法规名、课程名、指标名
4. 引文、书名、论文题名
5. 变量口径、数据口径

建议做成 `protected_items` 白名单。

规则：

1. 严重错误 `1` 个即失败
2. 轻微格式变化但不影响事实，可酌情扣分

### 6.6 语言质量与学术风格: 5 分

改写稿必须看起来像合格学术文本。

重点检查：

1. 是否通顺
2. 是否有错别字
3. 是否有明显病句
4. 是否口语化过重
5. 是否有机器拼接感

## 7. 升级评分等级

建议按下面等级判断候选版本：

### S 级

1. `pass@5 >= 90%`
2. 平均复检 AIGC 率 `<= 4.5%`
3. 原意保持均分 `>= 95`
4. 严重事实错误 `0`
5. 严重语病 / 错别字 `0`

### A 级

1. `pass@5 >= 80%`
2. 平均复检 AIGC 率 `<= 5.0%`
3. 原意保持均分 `>= 92`
4. 严重事实错误 `0`
5. 严重语病 / 错别字 `0`

### B 级

1. `pass@5 >= 60%`
2. 平均复检 AIGC 率 `<= 8.0%`
3. 原意保持均分 `>= 88`
4. 不允许严重错误，但允许少量轻微表达问题

### C / D 级

1. 无法稳定压到目标区间
2. 高风险表达仍然大量残留
3. 原意、事实或语言质量经常受损

主线升级要求：

1. 至少达到 `A`
2. 所有硬门槛全部通过
3. 与当前主线版本相比有明确提升

## 8. 训练思路

降 AIGC 训练的重点，不是“写得像真人聊天”，而是让文本脱离平台容易判定为 AIGC 的表达模式，同时保持学术文本质量。

### 8.1 资料分层

同一平台的资料，拆成 3 层：

1. 结果评分集
   - 有原文
   - 有改写稿
   - 有改写前后真实平台 AIGC 报告
2. 规则抽取语料
   - 有原文和改写稿
   - 但报告不完整或配对不全
3. 保护项语料
   - 用来抽取实体、数字、术语、引用等不可误改内容

### 8.2 先抽“高 AIGC 风险表达骨架”

训练重点应放在平台高风险表达特征，而不是泛化地乱改全文。

高风险骨架通常包括：

1. 过于整齐的三段式论证
2. 高密度抽象名词堆叠
3. 机械式“首先 / 其次 / 最后”链
4. 结论模板句反复出现
5. AI 风格过强的均匀长句和标准化连接
6. 概括性很强但细节承载不足的段落

### 8.3 改写顺序

建议按下面顺序处理：

1. 冻结保护项
2. 识别高风险表达骨架
3. 做结构性改写
4. 做局部表达自然化
5. 检查论证和结论是否偏移
6. 最后做语言质量清洗

### 8.4 平台必须分开训练

不同平台对 AIGC 风格的敏感点不同，不能一套规则直接共用评分结论。

要求：

1. 知网单独训练
2. 维普单独训练

基础改写能力可以共用，但评分、晋级、主线发布必须分平台。

### 8.5 双阶段评测

第一阶段：离线预筛

1. 检查保护项是否被破坏
2. 检查高风险表达骨架是否被消解
3. 检查是否出现病句、口语化、错别字、漏段

第二阶段：官方平台复检

1. 对通过预筛的候选版本送官方平台复检
2. 以官方平台报告为最终结果

原则：

离线预测只能做预筛，不能代替官方平台复检。

### 8.6 训练闭环

每轮训练都按下面闭环执行：

1. 从平台资料包整理新样本
2. 去重并标记样本质量
3. 生成或更新 benchmark
4. 训练候选改写包
5. 先跑离线质量门
6. 再跑官方平台复检
7. 记录每个样本改写前后 AIGC 率变化
8. 记录失败原因：
   - 没压下去
   - 原意偏了
   - 事实错了
   - 太像机器洗稿
   - 语言质量坏了
9. 只针对主要失败原因调参
10. 回归 benchmark

## 9. 禁止做法

后续训练中，禁止以下做法：

1. 为了降 AIGC 把文章写成口语作文
2. 为了降 AIGC 改掉核心论点和结论
3. 乱删重要论证和关键细节
4. 编造原文没有的新事实
5. 机械堆砌“更像人写”的语气词和废话
6. 只看单篇成功结果，不看整套 benchmark
7. 只看最终分数，不看文本质量

## 10. 当前结论

降 AIGC 算法包的核心评价顺序，必须固定为：

1. 官方平台复检 AIGC 率是否 `<= 5%`
2. 原意和事实是否保持
3. 高风险表达是否真的被消解
4. 语言质量和学术风格是否守住

这 4 条里，任何一条不成立，都不能算真正可升级的降 AIGC 算法包。
