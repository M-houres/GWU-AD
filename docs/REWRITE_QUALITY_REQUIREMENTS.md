# 降 AIGC 改写质量要求

参见：

- `docs/REWRITE_RESULT_EVAL_FRAMEWORK.md`
- `docs/REWRITE_ALGO_TRAINING_MEMORY.md`
- `docs/CNKI_REWRITE_SAMPLE_RULESET.md`

## 1. 核心原则

降 AIGC 不是“把文本洗成另一篇文章”，而是在保持论文可用性的前提下，消解官方平台容易判为 AIGC 的表达方式。

优先级顺序必须固定为：

1. 原意不变
2. 事实不坏
3. 论证不断
4. 文风不散
5. 在上述前提下，把官方平台 AIGC 率压低

结论：

如果某种写法虽然能把 AIGC 率降下来，但会让文章变空、变假、变口语、变机械，那这种写法不能进入正式算法包。

## 2. 什么叫“乱写乱改”

后续评估中，下面这些情况都视为“乱写乱改”：

1. 研究对象被改掉
2. 核心论点被改偏、改弱、改反
3. 原本有依据的论证被改成空话
4. 数据、实体名、政策名、术语被改错
5. 原本连贯的段落被拆得支离破碎
6. 学术文本被改成明显口语文风
7. 为了显得“像人写的”加入废话、语气词、无关句
8. 前后段风格不统一，像拼接稿
9. 摘要、正文、结语表达不再一致
10. 文本看起来像机器洗稿，而不是正常学术改写

## 3. 必须保留的内容

### 3.1 研究层

1. 研究对象
2. 研究主题
3. 研究范围
4. 研究结论
5. 主要观点

### 3.2 事实层

1. 年份、时间、比例、金额、样本量、面积
2. 人名、校名、机构名、公司名、地名
3. 政策、法规、标准、课程名称
4. 数据口径、变量口径、样本口径

### 3.3 学术层

1. 专业术语
2. 书名号内标题
3. 引文信息
4. 图表、模型、指标名称

### 3.4 结构层

1. 摘要
2. 关键词
3. 引言
4. 正文章节结构
5. 结语 / 结论
6. 参考文献

## 4. 允许的改写方向

降 AIGC 允许改写，但应坚持“结构去模板化优先、词级替换靠后”的顺序。

### 4.1 允许做的事

1. 调整句内逻辑展开顺序
2. 将过度整齐的模板句重写为更自然的论证句
3. 把抽象名词堆叠改成更清楚的谓语表达
4. 将过于均匀的长句拆解重组
5. 补足必要的因果、范围、目的关系
6. 对机械化连接词链做自然化处理
7. 对概括过强、细节过弱的段落做适度充实
8. 保持学术文风前提下增强表达起伏

### 4.2 推荐优先做的事

1. 先处理高 AIGC 风险段
2. 先处理模板化骨架
3. 先保证结论和论证不偏
4. 最后再做语言润色

## 5. 不允许的改写方式

### 5.1 禁止型

1. 为了降 AIGC 改掉原文结论
2. 为了降 AIGC 编造新事实
3. 为了降 AIGC 删除关键论据
4. 把学术表达改成明显生活化口语
5. 大量堆砌“其实、比如、我们可以看到”等水词
6. 用空泛扩写替代真实改写
7. 保留原骨架只换几个词
8. 把概念句改得模糊不清
9. 误改专有名词、引文、术语
10. 前后风格分裂，像多版本拼接

### 5.2 高风险型

这些做法不是绝对禁止，但默认高风险，必须谨慎：

1. 大幅增删段落
2. 改动章节标题
3. 重写摘要和结语的判断句
4. 重写定义句和概念句
5. 改动带数字、比较关系、范围边界的句子

## 6. 合格改写应具备的文本特征

一个合格的降 AIGC 改写稿，读起来应该像“作者重新组织和润色了自己的学术表达”，而不是“被强行洗过一遍的残稿”。

应具备以下特征：

1. 主题清楚
2. 逻辑自然
3. 语言顺畅
4. 学术感稳定
5. 前后文风一致
6. 论证仍然扎实
7. 没有明显机器拼接感

## 7. 改写质量检查清单

每篇改写稿至少检查下面 8 项：

1. 研究对象和结论是否没变
2. 摘要、正文、结语是否仍然一致
3. 数据、术语、专名是否没错
4. 高风险模板句是否真的被消解
5. 是否仍保持学术语气
6. 是否存在病句、错别字、口语化过重
7. 是否存在机械扩写、废话增生
8. 是否存在为了降 AIGC 而硬改的明显痕迹

其中任一项明显失真，这篇改写就不能算合格结果。

## 8. 建议的改写强度控制

降 AIGC 不是改得越狠越好。

建议控制原则：

1. 优先重写高风险段，不做无差别全文乱改
2. 优先消解模板结构，不优先动实体词
3. 优先让表达更自然，不优先让字面变化更大
4. 能通过结构和节奏变化解决的，不靠无意义扩写解决
5. 能保持原学术风格的，不往口语风偏

## 9. 训练落地要求

后续算法训练时，这份文档要直接转成质量门：

1. 保护项识别
2. 禁止型写法拦截
3. 结构一致性检查
4. 学术风格回正
5. 语言质量清洗

也就是说，降 AIGC 算法包不能只做“改”，还必须做“护”：

1. 护原意
2. 护事实
3. 护论证
4. 护学术风格

## 10. 当前要记住的结论

后续不要把降 AIGC 理解成“越不像原文越好”。

正确理解应该是：

1. 官方平台复检率要低
2. 但文本质量不能掉
3. 如果只能靠乱改才能降下去，说明算法包能力还不够，不是文章该为算法让路
