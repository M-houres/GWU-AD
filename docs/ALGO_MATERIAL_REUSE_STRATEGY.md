# 算法训练资料复用策略

更新日期：2026-04-10

## 1. 核心原则

训练资料不能只按“一个文件只服务一个算法包”去看。

更合理的做法是：

1. 同一份资料先按 `平台` 分桶。
2. 再按 `用途层级` 切分，而不是只按 `功能名` 切分。
3. 同一份资料可以同时进入多个训练池，但进入不同训练池时，角色和权重不同。

也就是说：

- 不混平台评分
- 但允许跨功能复用文本、规则、片段、结构特征

## 2. AIGC检测 和 降AIGC 的关系

二者不是独立关系，而是强耦合关系。

### 2.1 AIGC检测负责回答

1. 哪些段落容易被平台判成 AI 特征显著
2. 哪些句式、结构、连接方式会被放大风险
3. 高、中、低风险片段通常集中在哪些段落位置
4. 平台更在意哪些“模板感”“匀质感”“异常改写痕迹”

### 2.2 降AIGC负责回答

1. 如何改掉这些高风险表达
2. 哪种改法最容易把官方平台结果压到 `5%` 以内
3. 哪些改法虽然降了分，但会把文章改坏

所以本质上：

- `AIGC检测` 提供的是“判别规则”
- `降AIGC` 提供的是“逆向改写策略”

前者决定“该打哪里”，后者决定“该怎么改”

## 3. 同一份资料的多用途拆解方式

以 `知网 AIGC 全文报告` 为例，一份资料至少可以拆成四层价值：

### 3.1 结果层

可提取：

- 总体 AIGC 比例
- 显著 / 疑似 / 未标识占比
- 片段分布
- 段落高亮强弱

适用：

- `cnki_aigc_detect` 结果拟合

### 3.2 正文层

全文报告往往自带 `原文内容`。

虽然它不如原始 `docx` 干净，但仍可作为：

- `弱正文源`
- 段落切分源
- 句段风险映射源

适用：

- `cnki_aigc_detect`
- `cnki_rewrite`

### 3.3 规则层

从高风险段落中抽取：

- 模板连接词
- 段首重复形式
- 摘要、引言、结论等区块的高风险展开方式
- 机器改写痕迹
- 容易触发“显著”的局部句式

适用：

- `cnki_aigc_detect`
- `cnki_rewrite`
- 部分可迁移到 `cnki_dedup`

### 3.4 逆向改写层

如果同题后续又补进改写稿，或同目录已有润色前后稿：

- 可把“高风险原句 -> 较低风险改写句”沉淀成局部改写规则
- 可反推哪些改写是有效的，哪些只是表面换词

适用：

- `cnki_rewrite`

## 4. 资料用途分层

以后统一分成三层，不再简单分“能用 / 不能用”。

### A层：严格评分样本

要求：

1. 平台明确
2. 原文明确
3. 官方结果明确
4. 标题和版本对应关系明确

用途：

- 进入 benchmark
- 参与升级评分

### B层：弱监督样本

要求：

1. 官方全文报告存在
2. 可从报告里回抽正文
3. 总分、片段、高亮可提取
4. 但原始 `docx` 或版本关系不完全干净

用途：

- 规则抽取
- 片段监督
- 预评分集
- 候选 benchmark 池

### C层：规则素材

要求：

1. 只有正文
2. 或只有简洁报告
3. 或只有改写稿
4. 或存在对应关系不明确

用途：

- 规则池
- 提示词池
- 改写模式池

不能直接用于严格升级评分

## 5. 平台内可复用，平台间不混评分

必须坚持：

1. `cnki` 的评分集只评 `cnki` 包
2. `vip` 的评分集只评 `vip` 包
3. `paperpass` 的评分集只评 `paperpass` 包

但可共享的东西包括：

1. 文本清洗方法
2. 全文报告回抽正文方法
3. 章节识别方法
4. 句段切分方法
5. 风险片段抽取器
6. 改写质量约束器

所以：

- `评分标准` 分平台
- `提特征工具` 可共享
- `局部语言模式` 可参考迁移

## 6. AIGC检测、降AIGC、降重复率的复用关系

### 6.1 AIGC检测 -> 降AIGC

直接强相关。

可复用：

1. 高风险段落
2. 高风险句式
3. 风险区块分布
4. 高亮片段

### 6.2 降AIGC -> AIGC检测

也反向有价值。

可复用：

1. 有效改写样本
2. 无效改写样本
3. “改前高分 / 改后低分”的局部对照

这些能反过来告诉检测器：

- 哪些表述最敏感
- 哪些词法变化不影响平台判断
- 哪些结构变化会显著改变平台结果

### 6.3 查重资料 -> 降AIGC

不是直接同类，但有交叉。

可复用：

1. 模板化表达
2. 机械连接段
3. 同义替换伪改写
4. 高重复句法骨架

这些对 `降AIGC` 有辅助价值，但权重应低于真实 AIGC 样本。

## 7. 以后训练时的标准动作

每次新增资料后，统一按下列顺序处理：

1. 先判断平台
2. 再判断资料类型
   - 原始正文
   - 全文报告
   - 简洁报告
   - 改写稿
   - 压缩包
3. 再判断可进入哪一层
   - A层严格评分样本
   - B层弱监督样本
   - C层规则素材
4. 最后做跨功能复用
   - 检测
   - 降AIGC
   - 降重复率

## 8. 当前应采用的训练思路

基于现有资料，当前最合理的思路不是“平均训练 9 个算法包”，而是：

1. 先把 `cnki_aigc_detect` 做强
   - 因为现在知网 AIGC 结果资料最密集
2. 再把 `cnki_rewrite` 与其联动
   - 直接复用高风险句段、结构特征、异常改写痕迹
3. `cnki_dedup` 同步吃查重全文报告
   - 但评分口径保持独立
4. `vip_dedup` 继续维持独立推进
5. `vip_aigc_detect / vip_rewrite / paperpass_*` 等待更完整资料

## 9. 当前要记住的结论

1. 全文报告不仅是“结果文件”，也是“弱正文源”。
2. AIGC检测和降AIGC必须联动训练，不能各练各的。
3. 平台之间不能混评分，但平台内不同功能之间可以共享文本特征和局部规则。
4. 以后资料处理要从“单用途”升级为“分层复用”。
