# 小程序端到端测试清单

## 0. 前置准备

1. 后端服务可访问，`/api/v1/health` 正常
2. 小程序 `config/env.js` 指向测试或生产后端
3. 后台已开启 `wechat_miniprogram_login_enabled`
4. 支付配置完整，正式环境可拉起真实微信支付
5. 微信公众平台已完成“用户隐私保护指引/隐私声明”配置，并覆盖：
   - 微信手机号快捷验证组件 `getPhoneNumber`
   - 从微信聊天记录选择文件 `chooseMessageFile`
6. 真机微信中先确认隐私授权弹窗可正常拉起；若出现 `privacyagreement`，先修平台配置再继续测代码

## 1. 登录链路

1. 首次打开进入登录页
2. 勾选服务协议与隐私政策
3. 点击“微信隐私授权”并确认成功
4. 点击微信手机号快捷登录，成功进入首页
5. 退出后再次登录成功
4. 异常场景：
   - 关闭小程序登录配置后，登录应提示不可用
   - 使用过期 token 请求时，自动跳回登录页
   - 如果 `getPhoneNumber` 首次因隐私授权被拦，页面应提示“请再点一次登录”
   - 如果出现 `privacyagreement`，应判定为微信公众平台隐私声明未配置完整，不继续归咎前端代码

## 2. 任务提交链路

1. 首页分别提交：
   - AIGC 检测
   - 学术润色
   - 降重复率
2. 每个任务都覆盖：
   - 从微信聊天记录选择文件
   - 必填项完整填写
   - 提交成功后可跳转记录页
3. 异常覆盖：
   - 若 `chooseMessageFile` 报 `fail api scope is not declared in the privacyagreement`，应判定为微信公众平台隐私声明缺项
   - 上传超时或网络中断后，检查记录页是否已创建任务
   - 非法文件格式是否被前端拦截
4. 记录页检查：
   - 状态从 `pending / running` 进入 `completed / failed`
   - 失败任务能看到错误信息
   - 已完成任务可下载并打开结果文件

## 3. 支付链路

### 3.1 微信支付正式链路

1. 在“我的”页选择微信支付并创建订单
2. 成功拉起 `wx.requestPayment`
3. 支付完成后返回小程序，订单状态更新为已支付
4. 个人中心积分增加

### 3.2 测试 / 兜底链路

1. 测试模式下使用 `mock` 渠道创建订单
2. 点击“我已支付”后订单成功入账
3. 订单状态轮询可从 `created` 进入 `paid / closed`

### 3.3 支付异常链路

1. 取消支付：提示“你已取消支付”
2. 支付失败：提示“支付未完成，请重试”或通用失败提示
3. 订单超时：状态变为 `closed`

## 4. 记录与详情

1. 记录页状态文案统一显示
2. 卡片展开后可直接查看摘要指标和结构化结果
3. 失败任务显示错误信息
4. 非完成状态点击下载会被拦截并提示

## 5. 兼容与性能

1. 机型覆盖：iOS + Android
2. 网络覆盖：Wi-Fi、4G、弱网
3. 并发覆盖：连续提交任务 + 同时轮询订单
4. 关注指标：
   - 首屏可交互时间
   - 列表加载时间
   - 轮询稳定性

## 6. 回归通过标准

1. 核心链路全部通过：登录、提交、处理、下载、支付、积分入账
2. 不出现阻塞级报错：白屏、崩溃、支付无法拉起
3. 文案一致、状态一致、失败提示可理解
